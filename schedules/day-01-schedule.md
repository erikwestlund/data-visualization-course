# Day 1 Schedule: Workflow And First Visualization

Assumed class block: 3 hours.

## Main Goal

Students leave with the course project working, `source("updater.R")` tested, and one first visualization module opened or started.

## Schedule

| Time | Segment | Materials | Notes |
|---:|---|---|---|
| 0:00-0:10 | Welcome and course framing | `slides/day-01-intro-and-workflow.qmd` | Keep this brief. |
| 0:10-0:25 | Introductions | Slides or verbal prompt | Pair intros are faster than full-room intros. |
| 0:25-0:50 | Course philosophy and workflow preview | Day 1 slides | Do not over-teach grammar yet. |
| 0:50-1:35 | Course materials walkthrough | `modules/01-workflow-and-basics/01_course-materials.qmd` | Main event. Go slowly. Everyone clicks along. |
| 1:35-1:45 | Break and tech catch-up | RStudio/Positron help | Use this as setup buffer. |
| 1:45-2:25 | First visualization demo | `modules/01-workflow-and-basics/02_first-visualization.qmd`; references: `modules/01-workflow-and-basics/03_ggplot2-dictionary-and-cheatsheet.qmd`, `modules/01-workflow-and-basics/04_dplyr-tidyr-dictionary-and-cheatsheet.qmd` | Emphasize `p1`, `p2`, `p3` object workflow and point students to the cheatsheets. |
| 2:25-2:55 | Student practice | `practice/templates/01-workflow-and-basics/01_explore-and-visualize.qmd` copied to `practice/work/` | Goal is one `glimpse()`, one count, one plot attempt. |
| 2:55-3:00 | Wrap and next steps | Verbal | Remind: work in `practice/work/`. |

## If Time Runs Short

Cut slide depth, not workflow. The minimum successful Day 1 is project open, updater tested, and first plot started.

## Instructor Priorities

- Confirm project root and paths.
- Confirm `source("updater.R")` works.
- Confirm `practice/work/` exists.
- Confirm students understand `modules/` versus `practice/work/`.
- Confirm students know where to find the `ggplot2` and `dplyr`/`tidyr` dictionary modules.
